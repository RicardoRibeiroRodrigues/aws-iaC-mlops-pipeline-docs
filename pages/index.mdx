# Introdução

Neste repositório está contida a documentação criada para o projeto [pipeline mlops iac](https://github.com/RicardoRibeiroRodrigues/AWS-IaC-mlops-pipeline), para que seja possível replicar fácilmente a infraestrutura criada nele, assim como facilitar modificações e seu uso.

## Para que serve este projeto ? 

Esse serviço tem como base a expansão da importância da integração contínua e da entrega contínua (CI/CD), o primeiro tem como seu objetivo encontrar e investigar erros mais rapidamente, melhorar a qualidade do código, além de reduzir o tempo necessário para validar e lançar novas atualizações de software, já é a automatização da criação, teste e preparação do código para produção quando alterado, essa prática estende da integração contínua, implantando as alterações em ambientes de teste e/ou ambiente de produção. Esses dois juntos são parte fundamental da área de DevOps, que vem como uma solução de automatizar deploys, que antes eram manuais e demorados, além de contribuir para identificar bugs com antecedência e criar código de maior qualidade.
Além do DevOps tradicional, foi criado com o avanço das tecnologias de Inteligência Artificial, o MlOps, que segue a mesma lógica do DevOps, só que para o desenvolvimento de serviços e modelos relacionados a Machine Learning. A maior diferença se dá no de que no MlOps, o modelo deve ser treinado no pipeline, e o teste deste é um pouco mais complexo que os testes de softwares mais “tradicionais”.

Ferramenta da aws para automatizar pipelines de entrega contínua, oferecendo atualizações rápidas e confiáveis de aplicações e infraestruturas. Um exemplo de pipeline é a partir de uma mudança no código fonte (em um commit no git por exemplo), você ter um código que automaticamente compila a nova versão, roda os testes, e caso tudo corra bem nesse processo, você prepara a nova versão para deploy.
